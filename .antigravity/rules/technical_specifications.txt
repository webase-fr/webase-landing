3) Design system (règles non négociables)
3.1 Tokens OKLCH (dark mode impeccable)
Couleurs définies en OKLCH pour des accents vibrants en dark mode (sans “baver”).
Définir : --bg, --surface, --text, --muted, --border, --accent, --accent-2, --focus.
Dark mode system-first :
Par défaut : respecter prefers-color-scheme
Toggle optionnel (stockage localStorage) sans casser le mode système
3.2 Typo premium
2 fonts max :
1 display variable (titres)
1 text variable (corps)
Taille fluide via clamp() (pas de paliers abrupts)
Interlignage généreux + tracking léger sur les titres
3.3 Composants “signature”
Button : 1 style principal + 1 secondaire (outline), avec micro-anim (shine/translate 1–2px)
Card : surfaces avec border subtile + shadow soft + highlight au hover
Section : spacing cohérent (mobile/desktop), séparateurs hairline en dégradé
Bento grid sobre (pas “SaaS cliché”), beaucoup d’air, alignements stricts

4) Lenis (smooth scroll) — spec d’implémentation
Objectifs
Scroll fluide “premium” sans casser l’accessibilité ni les ancres.
Compatible animations (reveal) + navigation.
Règles
Lenis activé uniquement côté client (provider use client).
Ancres : quand on clique un lien #section, utiliser la méthode Lenis pour scroller proprement.
Respecter prefers-reduced-motion :
Si l’utilisateur préfère réduire les animations → désactiver Lenis et revenir au scroll natif.
Sources Lenis + exemples d’intégration Next : (lenis.darkroom.engineering)

5) 3D subtile (R3F) — spec “pro” (pas gadget)
Objectifs
Montrer maîtrise technique en 3 secondes, sans gêner la lecture.
Ultra léger, pas de scène lourde.
Règles
Un seul élément : globe abstrait / forme paramétrique / “wireframe orb”.
Canvas contenu dans Hero (ou entre Hero et Offres), taille limitée, opacity maîtrisée.
Interactions : rotation très douce au mouvement souris (amplitude faible).
Perf :
dpr={[1, 1.5]} (cap)
frameloop="demand" si possible (ou throttling)
éviter postprocessing lourd
Fallback :
Sur mobile bas de gamme / prefers-reduced-motion → remplacer par SVG statique.
Docs R3F + exemples : (r3f.docs.pmnd.rs)
Starter “DOM + Canvas” (pattern utile) : (GitHub)

Règles de design (pour éviter le rendu “classique”)
Pas de dégradés arc-en-ciel : seulement du halo “brand” très doux + un accent secondaire rare.


Hairlines (1px) + shadows soft : aspect premium.


Contrastes maîtrisés : texte jamais 100% blanc partout (utiliser --text-muted).


Un seul élément “wow” : le 3D + halo. Tout le reste reste sobre.


Focus states visibles (expertise + accessibilité).
